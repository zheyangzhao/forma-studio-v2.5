# Claude 巡邏：v2 程式邏輯醫療偏向稽核

日期：2026-04-30
方法：grep + sed 找 v2.html 所有規則邏輯、預設值、system prompt 中的醫療偏向。
角度：程式邏輯（不只文案）— 為什麼紫微斗數老師輸入「紫微入門課程」會被分到醫療？

---

## §一、致命問題：Smart 的 GPT system prompt（line 1079）

```js
content:`你是 O2Win 牙醫診所的 AI 助理，幫使用者判斷需求。
根據使用者描述，回傳 JSON：
{
  "nlm_task":"簡報製作|文件整理|文獻分析|圖片提示詞|教學內容|自由提問",
  "nlm_domain":"口腔醫學|醫療生技|學術研究|診所經營|書稿寫作|知識管理",
  "nlm_aud":"成人患者/家長|牙醫同行|學生/實習醫師|兒童患者",
  "design_imgStyle":"醫學教科書|溫暖手繪|資訊圖表|極簡線條|3D 寫實|攝影寫實",
  ...
}`
```

**用戶輸入「紫微斗數入門課程」會發生什麼？**
1. GPT 被指令當成「O2Win 牙醫診所 AI 助理」
2. JSON schema 強迫 nlm_domain 只能六選一：**全部醫療相關**（口腔/醫療生技/學術/診所/書稿/知識管理）
3. nlm_aud 只能選「成人患者/家長 / 牙醫同行 / 學生/實習醫師 / 兒童患者」 — **全部醫療**
4. design_imgStyle 第一個選項就是「醫學教科書」

**結論**：紫微斗數老師的需求會被 GPT 強迫塞進醫療框架。**這是最根本的偏向**。

決策摘要：Smart 的 system prompt + JSON schema 定義了使用者邊界，紫微/商業/教育用戶完全沒有合適選項。

## §二、本地規則 fallback 也偏醫療（line 1547-1559）

```js
const domain = /tmj|顳顎|咬合|矯正|牙|口腔|磨牙|睡眠|鼾/i.test(combined)?'口腔醫學'
  : /書|寫作/i.test(combined)?'書稿寫作'
  : /學術|文獻|論文/i.test(combined)?'學術研究'
  : /診所|管理|經營/i.test(combined)?'診所經營'
  : '口腔醫學';   // ← FALLBACK 直接到醫療

const audience = /患者|家長|大眾|病人/i.test(combined)?'成人患者/家長'
  : /同行|醫師|牙醫/i.test(combined)?'牙醫同行'
  : /學生|實習/i.test(combined)?'學生/實習醫師'
  : isTeach?'學生/實習醫師'
  : '成人患者/家長';   // ← FALLBACK 直接到患者
```

**用戶輸入「紫微斗數入門課程」（API key 未設、走本地規則）**：
1. 沒命中口腔/牙/書/學術/診所 → fallback `'口腔醫學'`
2. 沒命中患者/同行/學生 → fallback `'成人患者/家長'`
3. 結果：紫微斗數老師看到自己被分到「口腔醫學 + 成人患者/家長」

決策摘要：本地規則對非醫療關鍵字完全沒處理，全部 fallback 到醫療。
