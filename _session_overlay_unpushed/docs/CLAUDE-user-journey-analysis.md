# Claude 視角：使用者旅程審視（v2.0 七 tab 整併性分析）

日期：2026-04-30
方法：Playwright 實際操作 v2，從零模擬 3 個典型使用者旅程，記錄路徑斷裂點與冗餘。
角度：使用者體驗（不看程式碼，只看「我是用戶會怎麼用」）。

---

## §一、3 個旅程的實測結果

### 旅程 1：「我想做一張植牙衛教海報」

**走 Smart 一個 tab**：
| 步驟 | 動作 | 結果 |
|---|---|---|
| 1 | 進入 v2 預設停留在 🎨 Claude Design | 不確定該用哪個 tab |
| 2 | 點 🖼️ 智慧製圖 | 看到「說明您要做什麼」textarea + 6 個快速範例 |
| 3 | 輸入「做一張植牙衛教海報，A4 直式，給患者看的，要簡單易懂」 | 自動偵測完成 |
| 4 | 看到 AI 自動建議：工具=圖像生成 / 平台=GPT Image 2 / 風格=醫學教科書（Gray's Anatomy） / 解析度=2K 4 張 | 全部正確、不需修改 |
| 5 | 點「✨ 確認，生成提示詞」 | 產出完整英文 prompt |
| 6 | 完成卡 3 步驟 + 「→ 前往 Claude Design 精修」按鈕 | 旅程結束 |

**Smart 一個 tab 解決 90%** — 自動偵測「口腔醫學關鍵字」「患者/家長受眾」「圖片需求」三件事，自動推論到正確的工具+平台+風格。

**沒用到的 tab**：Style Studio（Smart 已選風格）、Prompt Lab（Smart 已直接產 prompt）、Audit（Smart 結果已可用）、Claude Design（精修才用）、NotebookLM、彩蛋。

### 旅程 2：「把這篇牙周病文獻整理成可以在研討會報告的簡報」

**走 Smart 一個 tab**：
| 步驟 | 動作 | 結果 |
|---|---|---|
| 1 | 點 🖼️ 智慧製圖 | 同入口 |
| 2 | 輸入「把這篇牙周病文獻整理成可以在研討會報告的簡報」 | 自動偵測完成 |
| 3 | 看到 AI 自動建議：工具=NotebookLM / 任務=簡報製作 / 領域=口腔醫學 / 框架=IMRAD 實證 / 受眾=成人患者/家長 / 語言=正體中文+術語附英文 | 全部正確 |
| 4 | 點「✨ 確認，生成提示詞」 | 產出兩段 NotebookLM 智慧指令（YAML 參數 + 執行指令）|
| 5 | 完成卡：「貼到 NotebookLM 對話框 → 自定義指示貼設定頁」 | 旅程結束 |

**Smart 取代了 NLM tab 5 step 流程的 Step 1-4**：使用者不用點 11 任務 / 20 領域 / 12 框架的下拉。
NLM tab 變成「想精細控制每個選項」的進階使用者場景。

### 旅程 3：「我想瀏覽 116 條 prompt」（沒實際跑，但常用場景推測）

只走 Prompt Lab 一個 tab：搜尋、過濾、套用、收藏。獨立性高，難以與其他 tab 整併。

決策摘要：3 個典型旅程中，Smart 解決 80% 場景，Prompt Lab 是獨立資源庫，其他 5 個 tab 變成補強角色。

## §二、tab 之間的功能重疊（從用戶角度）

| 重疊組 | tab A | tab B | 重疊在哪 | 用戶感受 |
|---|---|---|---|---|
| **重疊組 1：設定圖像 prompt 的入口** | 🖼️ 智慧製圖 | 🎨 Claude Design | Smart 自動推論完整選項 vs Claude Design 4 區塊手動點選 | 兩個 tab 都讓我「設定要畫什麼」。Smart 簡單、Design 精細 |
| **重疊組 2：組裝風格的入口** | 🖼️ 智慧製圖 | 🎬 風格實驗室 | Smart 自動選風格 vs Style Studio 5 類別 chips 手動選 | Smart 已經幫我選了風格，我為什麼還要用 Style Studio？ |
| **重疊組 3：取得現成 prompt 的入口** | 🎬 風格實驗室 | 📚 提示詞庫 | Style Studio 組出新 prompt vs Prompt Lab 從 116 條挑 | 兩個都產 prompt，差別在「自己組」vs「現成挑」 |
| **重疊組 4：設定 NLM 任務指令** | 🖼️ 智慧製圖 | 📑 NotebookLM | Smart 自動推論 5 step vs NLM 手動 5 step | Smart 1 個 textarea + 確認 = 取代 NLM 的 Step 1-4 |
| **重疊組 5：分析既有 prompt 品質** | 🩺 體檢 & 增強 | （獨立）| 沒有重疊。Audit 的場景是「我有現成 prompt，想體檢」 | Audit 是獨立工具，不重疊 |
| **重疊組 6：彩蛋實驗** | 彩蛋 🧪 | 📑 NotebookLM | Lab 4 個實驗（病毒/TED/反 slop/批次）都引導去 NotebookLM | Lab 是「快速骨架」，正式產出走 NLM |

**總結**：7 個 tab 中至少 4 個 tab 在做類似的事（Smart / Claude Design / Style Studio / NotebookLM 都是「設定 prompt 的入口」）。

決策摘要：重疊組 1+2+4 共用 Smart 為自動入口、其他為手動精修；重疊組 3 可整併但功能不同（自組 vs 挑現成）；重疊組 5 獨立保留。

## §三、看診時真的會用哪些 tab？

從牙醫看診實際場景推測使用頻率：

| Tab | 使用頻率推測 | 理由 |
|---|---|---|
| 🖼️ 智慧製圖 | **高頻必用**（80% 場景）| 自然語言輸入直接產 prompt，看診忙時最快 |
| 🎨 Claude Design | 中頻（20% 場景）| 需要精修風格、品牌、設計哲學時才用 |
| 📑 NotebookLM | 中頻（15% 場景）| 想精細控制 11 任務 × 12 框架時才用 |
| 📚 提示詞庫 | 中頻（30% 場景）| 想找靈感、看別人怎麼寫、複製現成 prompt |
| 🩺 體檢 & 增強 | 低頻（5% 場景）| 已經有 prompt 想體檢時才用 |
| 🎬 風格實驗室 | **低頻（< 5%）** | 如果 Smart 自動選風格已經夠好，這個 tab 變雞肋 |
| 彩蛋 🧪 | 低頻（< 5%）| 純實驗工具 |

**潛在問題**：Style Studio 是 Sprint 3 才加的新功能，但實測下來 Smart 已經能做到 80%。Style Studio 對「不想精細控制但想看 chips 一目了然」的使用者有價值，但對牙醫看診忙的場景幫助小。

決策摘要：只有 🖼️ 智慧製圖 是高頻必用；🎬 風格實驗室、彩蛋 🧪、🩺 體檢 是低頻；其他是中頻精修。

## §四、我建議的整併方向（給 Codex 對比用）

### 方向 1：Smart 變主入口，其他變「精修工具」

把 Smart 從第 3 個 tab 移到**第 1 個 tab 預設位置**，標題改為「快速生成」。
其他 6 個 tab 維持但歸類為「進階工具」（用 sub-tab 或下拉選單收起來）。

**優點**：
- 80% 場景一個 tab 解決
- 進階使用者仍可用 Claude Design / NLM 精修
- v1.1 的 4 大 tab 全保留

**缺點**：
- 需要重排 tab 順序（潛在學習成本）
- v1.1 凍結契約限制：只能改 v2，但要說服自己 v1.1 順序是「過去式」不是「標準」

### 方向 2：合併重疊度最高的 3 對 tab

| 合併前 | 合併後 |
|---|---|
| 🎬 風格實驗室 | 併入 🎨 Claude Design 作為「精選風格」sub-tab |
| 📑 NotebookLM 5 step | 併入 🖼️ 智慧製圖 作為「精細模式」switcher |
| 彩蛋 🧪 4 個實驗 | 併入 🩺 體檢 & 增強 的「進階工具」區 |

7 tab → 4 tab：🖼️ 智慧製圖（含 NLM 精細）/ 🎨 Claude Design（含 Style Studio）/ 📚 提示詞庫 / 🩺 體檢 & 增強（含 Lab）

**優點**：
- 7 → 4 大幅簡化
- 每個 tab 都有清楚單一定位

**缺點**：
- 工程量大（重組 4 個 tab 的 React 元件）
- v1.1 凍結契約意義變弱（v2 與 v1.1 差異變大，使用者切換版本會迷惑）
- 可能毀掉現有的 cross-tab bridge（pendingApply 邏輯）

### 方向 3：用 wizard 把流程串起來，tab 只剩 2-3 個

| Wizard | 步驟（每步用對應功能）|
|---|---|
| 🚀 一鍵生成 wizard | Smart 自動偵測 → Audit 體檢分數 → AI 增強（選用）→ 套用到 Design 精修（選用）→ 複製生圖 |
| 📚 瀏覽 prompt | Prompt Lab（保留獨立）|
| 🛠 進階手動 | Claude Design / NLM / Style Studio / Lab 全部塞進這個 tab 的 sub-tab |

7 tab → 3 tab：🚀 一鍵生成 / 📚 瀏覽 prompt / 🛠 進階手動

**優點**：
- 主入口極度簡化（一個 wizard 走完整流程）
- 進階使用者仍可手動精修

**缺點**：
- 工程量極大（要寫 wizard 跳轉邏輯 + 整併所有現有 tab）
- v1.1 凍結契約完全脫節（v2 變成完全不同的產品）
- 可能失去使用者熟悉感

決策摘要：方向 1 最保守（重排順序）、方向 2 中度（合併重疊）、方向 3 大刀闊斧（wizard 化）。

## §五、Claude 個人傾向

如果是我（用戶），會選 **方向 1 + 部分方向 2**：

1. **把 🖼️ 智慧製圖移到 tab 0 預設位置**（取代 🎨 Claude Design 的預設地位）
2. **🎬 風格實驗室合併進 🎨 Claude Design** 作為「📦 風格組合」sub-tab（或乾脆移除）
3. **彩蛋 🧪 改為 🎨 Claude Design 的「進階」sub-tab** 不獨立 tab
4. 保留：🖼️ 智慧製圖 / 🎨 Claude Design / 📑 NotebookLM / 📚 提示詞庫 / 🩺 體檢 & 增強 = **5 tab**

**為什麼不走方向 3**：
- 工程量過大，v1.1 凍結契約失去意義
- 看診忙時，使用者習慣的「進入 Smart → 一鍵生成」就是 wizard 的精神，不需要強制 wizard layout
- Prompt Lab 116 條是獨立資源，wizard 化會破壞瀏覽體驗

**待用戶確認的兩個關鍵點**：
- 是否同意「Smart 應為預設首頁，因為它已經是 80% 場景的最佳入口」？`(待用戶確認)`
- 是否同意「🎬 風格實驗室移除或併入」？`(待用戶確認)`

決策摘要：Claude 推薦 7 → 5 tab，把 Smart 升為主入口，移除/合併 Style Studio 與彩蛋 Lab。
