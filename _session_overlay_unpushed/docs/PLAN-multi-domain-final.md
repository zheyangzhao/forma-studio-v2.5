## §一、用戶需求重述

> 「以一般人使用與醫療人員使用都方便為主，千萬不要侷限在醫療」
> 「我還要製作紫微斗數的課程，也是需要的」
> 「所以要多面向開發」

關鍵：用戶要的是 **N 領域支援**（不只「一般 + 醫療」），舉例「紫微斗數」是要測試「非主流領域」是否被照顧到。

決策摘要：解決方案不是「加領域選擇器 UI」，而是**讓 Smart 對任何領域的一句話輸入都正確分類**。

## §二、共識：兩份報告一致的關鍵發現

### 1. 致命問題：Smart 的 GPT system prompt 把使用者鎖在醫療
**位置**：`web/forma-studio-v2.html` line 1079
**內容**：
```js
content: `你是 O2Win 牙醫診所的 AI 助理...
{
  "nlm_domain": "口腔醫學|醫療生技|學術研究|診所經營|書稿寫作|知識管理",
  "nlm_aud": "成人患者/家長|牙醫同行|學生/實習醫師|兒童患者",
  "design_imgStyle": "醫學教科書|溫暖手繪|資訊圖表|極簡線條|3D 寫實|攝影寫實"
}`
```
- nlm_domain 六選一**全部醫療**
- nlm_aud 四選一**全部醫療**
- 紫微斗數老師輸入「紫微入門課程」會被 GPT 強迫塞入醫療框架

### 2. 本地規則 fallback 也偏醫療
**位置**：line 1547-1567（SmartTab.analyze）+ line 1033（ruleAnalyze）
- `domain` fallback = `'口腔醫學'`
- `audience` fallback = `'成人患者/家長'`
- `imgStyle` fallback = `'醫學教科書'`

### 3. Prompt 模板硬寫 O2Win 品牌名
12 處 hardcode（line 1612 / 1655 / 1693 / 1703 / 1708 / 1731 / 1883 / 2412 / 2670 / 2785 / 2915）

### 4. 20 哲學 useCase 至少 13 個含醫療詞
紫微斗數老師看會覺得「全部不適合我」

### 5. 12 框架太醫療化
PICO 是純醫療；缺命理 / 教學 / 商業專用框架

### 6. 不需要強制加領域選擇 UI
無感支援優先；領域 chip 是修正工具，不是進門門檻

決策摘要：兩份報告共識「Smart system prompt + 本地 fallback + 模板 = 多領域核心障礙」，先修 Smart 規則層，UI 層擴展為次要。

## §六、明確不做的事

| 不做 | 理由 |
|---|---|
| ❌ 不改 v1.1 (`web/forma-studio.html`) | 永久凍結契約 |
| ❌ 不刪除既有醫療 chips | 醫療專業也是核心用戶 |
| ❌ 不強制使用者每次先選領域 | 無感判斷優先 |
| ❌ 不在 P0 引入新 schema 欄位 | 避免結構大改 |
| ❌ 不刪除 PICO 框架 | 標示「醫療專用」即可 |
| ❌ 不把醫療專業降級為二等場景 | 平等支援 |
| ❌ 不為了多領域加新 tab | 6 tab 已是 B-lite 共識 |
| ❌ 不把所有領域塞進同一巨大 prompt | 維持 prompt 簡潔 |
| ❌ 不用「列出領域名稱」假裝完成多面向 | 規則邏輯才是真正關鍵 |

決策摘要：所有改動只在 v2 規則 + 文案層，不動架構、不動 schema、不刪功能。
